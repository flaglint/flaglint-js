Security and Trust

A concise summary of FlagLint's data handling, safety boundaries, and release posture.

Runs locally

FlagLint is a local CLI. It reads source files from the directory you specify, performs static AST analysis in process, and writes reports or transformed files to stdout or disk. It does not require a LaunchDarkly SDK key for scan or migration commands.

Guarded apply

flaglint migrate --apply requires a clean git working tree unless explicitly overridden. It rewrites only call sites where FlagLint can prove a local or configured imported OpenFeature client binding. Dynamic keys, detail evaluations, unknown fallbacks, and bulk calls remain manual review.

CI and provenance

The repository includes Node 20 and Node 22 CI checks, CodeQL, Dependabot, and a release workflow configured for npm Trusted Publishing/OIDC. Runtime support remains Node.js 20 or newer.

Security reporting

Report suspected vulnerabilities privately through GitHub Security Advisories. For unsupported patterns or false positives, use the unsupported pattern issue template.
