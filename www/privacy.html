Privacy

What this website and the FlagLint CLI do and do not collect.

Effective date: 29 May 2026

CLI data handling

The FlagLint CLI (flaglint scan, flaglint migrate, flaglint validate) reads source files from paths you specify on your local machine. It performs static AST analysis in-process and writes reports or transformed files to stdout or to a file path you provide.

The CLI does not transmit source code, flag keys, file paths, LaunchDarkly SDK keys, migration reports, or any other data to FlagLint-owned or third-party infrastructure. No outbound network connections are made during scan, migration, or validation.

Reports are local. Scan reports, migration plans, and validation SARIF files are treated as source-derived artifacts — they may contain file paths and flag keys from your codebase. They are stored wherever you write them and are not uploaded by the CLI.

Website data handling

Analytics and tracking scripts

flaglint.dev does not load analytics or tracking scripts. No JavaScript from Google Analytics, Plausible, Fathom, Mixpanel, Segment, or any similar analytics service is present on this website.

Cookies

flaglint.dev does not set cookies. The static pages served by this domain do not use cookies for any purpose. Your browser's developer tools should show zero cookies from flaglint.dev.

Third-party services

This website is hosted on Cloudflare Pages. Cloudflare processes network requests as part of hosting and CDN delivery and may collect access logs under Cloudflare's privacy policy.

Page fonts are loaded from Google Fonts (fonts.googleapis.com, fonts.gstatic.com). Your browser sends a request to Google's servers to retrieve font files; this request is subject to Google's privacy policy.

Links on this site point to GitHub (github.com) and npm (npmjs.com). Visiting those links is subject to their respective privacy policies. FlagLint does not receive data about your interactions with those third-party sites.

Contact

For privacy or security questions, use GitHub Security Advisories for suspected vulnerabilities, or GitHub Issues for general questions.
