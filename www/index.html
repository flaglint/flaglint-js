The Problem

Migration is hard to do safely.

🔍
You don't know what you have

Direct ldClient.variation() calls are scattered across your codebase. Before migrating, you need an accurate inventory — which call types, which files, which are dynamic keys, which need manual review.

⚠️
Mechanical rewrites break things

The LaunchDarkly and OpenFeature APIs differ in argument order, context shape, and detail result format. A careless codemod loses fallback values, drops context, or silently breaks await. FlagLint only rewrites what it can prove is safe.

🚦
Migration has no finish line

Without enforcement, new direct LD calls creep back in after the migration. flaglint validate --no-direct-launchdarkly gives you a CI gate that exits 1 the moment a direct LD evaluation call appears outside your bootstrap file.

How It Works

Scan, migrate, enforce — in three steps

01
Inventory your codebase
$ flaglint scan ./src
JSON Markdown HTML SARIF

Generates detailed reports in JSON, Markdown, HTML, or SARIF. Know exactly what LaunchDarkly Node.js SDK calls exist, where they are, and which are safely automatable.

02
Review and apply migration
$ flaglint migrate --dry-run

Generates reviewable before/after diffs with provider setup guidance. When ready, --apply rewrites only automatable call-sites — preserving flag key, fallback, context, and await exactly. Detail methods, dynamic keys, and bulk calls require manual review.

03
Enforce in CI
$ flaglint validate \
  --no-direct-launchdarkly

A CI gate that exits 1 whenever a direct LaunchDarkly evaluation call appears outside your provider bootstrap file. Migration stays complete — no new LD calls can land unreviewed.

Features

Safe, guarded migration to OpenFeature

🔍
AST-based scanning

Detects LaunchDarkly Node.js server SDK calls by AST, not grep. Resolves import aliases, CJS require forms, and custom wrappers.

📊
Four report formats

JSON for pipelines. Markdown for PRs. HTML for sharing. SARIF for GitHub Code Scanning.

🛡
Guarded apply

Rewrites only when an OpenFeature client binding is proven in the file. Never touches detail methods, dynamic keys, or bulk calls.

⚡
Manual-review flagging

Dynamic keys, detail methods, and unknown fallbacks are reported for manual review — never silently skipped or incorrectly rewritten.

🔄
Argument-preserving codemod

Flag key, fallback value, evaluation context, and await are preserved exactly. LaunchDarkly remains your provider.

🚦
CI enforcement

Exit 1 on any direct LD evaluation call outside the bootstrap file. Enforce migration completeness on every PR.

Install

Works with any Node.js TypeScript or JavaScript project

Install globally
$ npm install -g flaglint
Or use without installing
$ npx flaglint scan .

Supports Node.js 22+. Works with Express, NestJS, Fastify, or any Node.js TypeScript/JavaScript codebase using the LaunchDarkly server-side SDK.
