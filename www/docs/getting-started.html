Getting Started

Install FlagLint and run your first flag scan in under two minutes.

What is FlagLint?

FlagLint is a TypeScript CLI tool that statically analyzes JavaScript and TypeScript codebases for direct LaunchDarkly Node.js server SDK calls, detects flag debt, and generates reviewable OpenFeature migration plans.

LaunchDarkly remains your feature flag provider. FlagLint migrates your application code to call OpenFeature's vendor-neutral evaluation API — keeping LaunchDarkly wired in behind the OpenFeature provider interface.

Requirements

  • Node.js 20 or newer
  • A JavaScript or TypeScript project that uses the LaunchDarkly Node.js server-side SDK (launchdarkly-node-server-sdk)
ℹ️

FlagLint's scan and migrate commands currently support the LaunchDarkly Node.js server-side SDK. Browser SDKs, React SDKs, and other providers are outside the current scope and will be reported for manual review.

Installation

Install globally via npm:

bash 
$ npm install -g flaglint

Or run without installing using npx:

bash 
$ npx flaglint scan .

Step 1 — Scan your codebase

Run a flag scan to inventory every direct LaunchDarkly Node.js SDK call in your project:

bash 
$ flaglint scan ./src

Example output:

output 
✔ Scanning 142 files...
✔ Found 18 direct LaunchDarkly Node SDK calls across 6 unique flags (61ms)
⚡ 3 dynamic flag keys — manual review required
↳ 2 detail method calls — manual review required

→ Run flaglint migrate --dry-run for a reviewable migration diff

The scan produces a Markdown report by default. Use --format json, --format html, or --format sarif for other formats.

Step 2 — Preview the migration plan

Generate a reviewable before/after diff. No files are modified at this stage:

bash 
$ flaglint migrate --dry-run

The dry-run output includes inline provider setup guidance showing which packages to install and how to wire up the LaunchDarkly OpenFeature provider. Complete this one-time setup before proceeding to apply.

See OpenFeature Provider Setup for the full bootstrap instructions.

Step 3 — Apply automatable transformations

Once provider setup is complete and your working tree is clean, apply the migration:

bash 
$ flaglint migrate --apply
⚠️

--apply requires a clean git working tree. Commit or stash any changes before running with --apply. FlagLint refuses to write to a dirty tree unless you pass --allow-dirty.

Review the resulting diff and run your test suite before merging.

Step 4 — Enforce in CI

After migration, prevent new direct LaunchDarkly calls from landing in PRs:

bash 
$ flaglint validate --no-direct-launchdarkly \
    --bootstrap-exclude src/provider/setup.ts

This exits 1 if any direct LaunchDarkly evaluation calls are found outside the specified bootstrap file. See GitHub Actions CI for a full workflow example.

Configuration (optional)

Create a flaglint.config.json (or .flaglintrc) in your project root to customise scan behaviour:

json 
{
  "include": ["**/*.{ts,tsx,js,jsx}"],
  "exclude": ["**/node_modules/**", "**/dist/**"],
  "provider": "launchdarkly",
  "minFileCount": 0,
  "reportTitle": "My Project Flag Report"
}
