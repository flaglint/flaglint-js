flaglint scan

AST-based inventory of every direct LaunchDarkly Node.js server SDK call in your codebase. Produces reports in JSON, Markdown, HTML, or SARIF format.

Usage

bash 
flaglint scan [dir] [options]

[dir] defaults to the current working directory when omitted.

Options

Option Default Description
--format <fmt> markdown Output format: json | markdown | html | sarif
--output <file> stdout Write report to a file instead of stdout
--config <path> auto-detect Path to a config file (flaglint.config.json / .flaglintrc)
--exclude-tests — Exclude *.test.*, *.spec.*, __tests__/, tests/ from scan results
⚠️

Passing an invalid --format value exits with code 2. Valid values are json, markdown, html, and sarif.

Exit codes

CodeMeaning
0Scan completed — no staleness signals detected
1Scan completed — configured staleness/review signals present
2Invalid option or argument
130Interrupted by SIGINT (Ctrl+C)

Exit code 1 on staleness/review signals allows CI pipelines to surface configured review candidates without blocking builds. Use continue-on-error: true in GitHub Actions if you want the SARIF upload to succeed even when review signals are present.

Examples

Default Markdown report to stdout

bash 
flaglint scan ./src

JSON output piped to a file

bash 
flaglint scan --format json --output report.json

SARIF for GitHub Code Scanning

bash 
flaglint scan --format sarif --output flaglint.sarif

HTML report

bash 
flaglint scan --format html --output report.html

Example output (Markdown)

output 
✔ Scanning 142 files...
✔ Found 18 direct LaunchDarkly Node SDK calls across 6 unique flags (61ms)
⚡ 3 dynamic flag keys — manual review required
↳ 2 detail method calls — manual review required

→ Run flaglint migrate --dry-run for a reviewable migration diff

JSON record shape

Each flag usage in JSON output includes:

json 
{
  "flagKey": "checkout-v2",
  "isDynamic": false,
  "file": "src/services/checkout.ts",
  "line": 14,
  "callType": "boolVariation",
  "stalenessSignals": []
}

file is always relative to the scan root, making reports comparable across machines. stalenessSignals is an array of configured review signal strings when any are present.

How detection works

flaglint scan uses AST-based analysis — not regular expressions — to find LaunchDarkly Node.js SDK calls. The scanner:

  • Resolves import aliases and CJS require() forms
  • Identifies the LaunchDarkly client binding by import source, not variable name
  • Detects typed method calls: boolVariation, stringVariation, numberVariation, jsonVariation, and their *Detail variants
  • Flags dynamic keys (computed at runtime) as manual-review items without guessing their value
  • Counts unique flag keys for staleness, not raw call-site counts
