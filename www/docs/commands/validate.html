flaglint validate

Validate that your codebase complies with feature flag policy rules. Designed for CI enforcement after your OpenFeature migration is complete.

Usage

bash 
flaglint validate [dir] [options]

[dir] defaults to the current working directory when omitted.

Options

Option Default Description
--no-direct-launchdarkly — Exit 1 if any direct LaunchDarkly Node server evaluation calls are found
--bootstrap-exclude <glob> — Repeatable glob pattern; matching files are excluded from violation checks
--config <path> auto-detect Path to a config file

Exit codes

CodeMeaning
0All rules passed — no violations found
1Violations found (direct LD calls detected when --no-direct-launchdarkly is set)
130Interrupted by SIGINT (Ctrl+C)

Examples

Report usages without enforcing

Running without flags always exits 0 — useful for visibility without blocking a build:

bash 
flaglint validate

Enforce — exit 1 on any direct LD call

bash 
flaglint validate --no-direct-launchdarkly

Exclude a provider bootstrap file

bash 
flaglint validate --no-direct-launchdarkly \
  --bootstrap-exclude src/provider/setup.ts

Exclude all files in a provider directory

bash 
flaglint validate --no-direct-launchdarkly \
  --bootstrap-exclude "src/provider/**"

Example output

Pass output

output 
✔ validate --no-direct-launchdarkly: no direct LaunchDarkly evaluation calls found.
  Scanned 42 file(s).

Fail output

output 
✗ validate --no-direct-launchdarkly: 2 direct LaunchDarkly evaluation call(s) found.

  src/services/checkout.ts:42:8 — boolVariation("checkout-v2")
  src/services/pricing.ts:17:4 — boolVariation(dynamic key — manual review required)

These files must migrate to OpenFeature before this rule passes.
Run flaglint migrate --dry-run to review the migration plan.

Using in GitHub Actions

Add flaglint validate as a CI step to block PRs that introduce direct LaunchDarkly calls after your migration is complete:

yaml 
- name: Enforce OpenFeature migration
  run: |
    npx flaglint validate --no-direct-launchdarkly \
      --bootstrap-exclude "src/provider/setup.ts"

See GitHub Actions CI for a full workflow including SARIF upload to GitHub Code Scanning.
